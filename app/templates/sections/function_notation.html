What is a function? In the section on Basic Functions in Tables and Words we encountered some functions—so you could start to have some examples of what this is supposed to mean—but we need to make it more explicit.

As mentioned in the section on Basic Functions in Tables and Words, a function is not a number nor even a set of numbers. But functions are about both of those things. Indeed, functions are relationships between sets of numbers. In a word, functions are about relationships. How is the price of pizza related to the number of toppings you buy? How is the height of a corn plant related to the number of days its been growing? There are a number of ways you might answer those questions in English. But, in terms of math, the answer is a whole raft of information packaged in—a function.

So, functions are relationships. Or they are the way we mathematically encode and work with relationships between variables. But what are they, really? We can and should get more precise about the concept. First of all, note that we are talking about relating one variable to another or even one data set to another—just as we did in the section Basic Functions in Tables and Words. There is a simple way to capture a relationship between two sets of numbers, and that is to set up a pairing between numbers in the two sets. If you think about it, that's really what's happening in data tables, like the following:

This table, as discussed elsewhere, pairs the number of toppings you order with the resulting price of a pizza. The two numbers (one, a number of toppings and, the other, the price) that share a column are paired up. A mathematician would describe this as a set of ordered pairs.

Now, I bet you're familiar with the concept of an ordered pair in math. This is your \((x, y)\)-pair—which can be plotted on the \((x, y)\)-plane. (We'll do that in later sections, as should come as no surprise.) We just write them with parentheses, with the different positions of the numbers having a special significance. (Later, when we graph them, we use the first coordinate or number to tell us how far and in what direction to travel horizontally and we use the second number to tell us how far and in what direction to travel vertically from the origin on the \((x, y)\)-plane.) For example, \[ (2, 3) \] is an ordered pair, but because of its "orderedness" (if that's a word) we have that \[ (2, 3) \not = (3, 2) \] Those are distinct mathematical objects.

So, back to functions: Functions are about defining or denoting a pairing of numbers between two sets (as is achieved in a data table). So, it should come as no surprise that functions have something to do with ordered pairs. In fact, they are (special kinds of) sets of ordered pairs.

Next question: Have you ever studied "set notation"? Well, since functions are sets, set notation (a way of writing down sets) can be useful to convey the true, mathematical meaning of what the word "function" is. So, let's review it here. First of all, set notation uses "curly braces", as in '\(\{\)' and '\(\}\)'. The set of objects that gets enclosed within these curly braces is called, well, a "set". For instance, you can have a set of numbers (if you like), like so: \[ \{1, 2, 3, 4\} \] That's a set! Pretty boring so far. Now, you can also choose symbols to represent these sets. Why not? (So, variables don't have to always represent numbers; they can be used to symbolize any object in the mathematical universe of concepts.) So, I could write \[ S = \{1, 2, 3, 4\} \] and I'd just be saying that I want to use the letter \(S\) to represent that set. Now, functions are sets of ordered pairs. No problem! Just as a letter (variable) can represent any mathematical object you want, you can also have a set of any sorts of mathematical objects you want. You could have a set of numbers, a set of ordered pairs, a set of triangles (e.g.: the set of all triangles congruent to a particular one), or even a set of sets! Actually, there are some pretty interesting things to think about when you start going down the road of "full mathematical generality", but all we need to look at are some sets of ordered pairs. Like this one: \[ \{(3, 12),(-4, 6),(7, 3),(4, 0)\} \] Is there any significance to the numbers I just chose? No. Is there any significance to the order in which I listed the different pairs? No, again: while there is any order to, well, ordered pairs, there is no order to sets. In fact, \[ \{1, 2, 3, 4\} = \{1, 3, 4, 2\} \] The two sets are equal! They refer to the same object.

There is one more thing worth mentioning here about how sets work: repeated elements in a set are redundant. This makes mathematical sets a little different from sets in the real world. For instance, you might have "duplicate" Mike Trout cards in your set of baseball cards, and you might feel pretty good about it. But there aren't multiple copies of mathematical objects floating around, in a sense. When we say "\(1=1\)" we mean that there is only one number 1! Thus, this is the kind of thing that happens when you try to put duplicates in a set: \[ \{1, 1, 1, 1, 1\} = \{1\} \] and \[ \{1, 1, 2, 3, 4, 4, 4\} = \{1, 2, 3, 4\} \]

That's enough said about sets as a general concept. But we need to say a few more words about sets of ordered pairs. Sets of ordered pairs are called relations. So, since a function is a set of ordered pairs, a function is a relation. So, what? We're just playing with words so far. Now for the important part: a function is a special set of ordered pairs—a special type of relation. The key is to realize the true purpose of functions.

The True Purpose of Functions

Functions as we have said too many times already are to capture or define a relationship between two variables or two data sets. But they are really for only one special type of relationship: "input/output relationships". For instance, your input could be "We want a 5-topping, medium pizza." Your output would be the price. Or, your input could be "I'm going to wait 2 months for my corn to grow." Your output might then be, "My corn is now 5 feet tall." Perhaps more interestingly, your input might be "We are going to buy 10,000 pounds of concrete; 1,000 yards of rebar; 5,000 man-hours; etc."—all at particular prices (the prices would also part of the input) and your output might be the profit you hope to make on a construction venture. That's the basic idea. On the other hand, there is something troubling about the real-world and these scenarios that we have considered here coming from it: the real world is kind of unpredictable. While it is certainly a worthy thing to try to contend with uncertainty—and there is certainly some mathematics out there that can help you to do that (take a course in probability or statistics if you want to know more in that direction)—at this stage, we are only going to consider a much "cleaner" or narrower point of view. Namely, the "input/output" relationships we are going to consider come with a special stipulation: for any given input, there shall be only one possible output—not two or three or more. For instance, if your order 5 toppings for your (medium) pizza, you will know full well what your bill will be. So far so good! On the other hand, it will be more a bit of pretend (word problems are good at pretending) to say that just because 2 months have gone by, we know that our corn will be exactly 5 feet tall. We don't know that at all! The problem with the corn (and with so many things) is that there are just too many variables. You see, that's actually the conceit of many when in comes to mathematics: If only we knew the values of all the variables that could come into play, then maybe we could make a better prediction? But enough about that. The point is that you have to start somewhere, so we start with the following very simple concept. I give you the official definition of function.

The official definition of function

A function in mathematics is a set of ordered pairs in which any first coordinate appears only once.

That may seem very abstract—and so it is! But here is the point: with a set set of ordered pairs that we are calling a "function", we regard the first coordinate as the input. We regard the second coordinate as the corresponding output. So the definition above really says that "for any given input, there will always be just one output, not many." In other words, when it comes to functions, there is no uncertainty. To use a 75-cent word: functions are deterministic: the input determines the output.

It won't say one thing on Tuesday and another on Wednesday. No! The square of 423 is the same thing no matter what.)

Now, just as the calculator labels its functions—so you know which ones it will execute when you tap the button with that symbol—we weill also label our functions with symbols. We will use variables (by which I just mean, letters) to label them. Well, actually, at a later stage, we will even use a series of letters to represent a function. Some common letters use to represent functions are \[ f, g, h \] However, some other import symbols (names) for functions that have more than one letter in them—which we will learn about in this and future courses and which are marked on your calculator!—are \[ \log, \ln, \sin, \cos, \tan \] So, it is by no means a rule that only one letter can be used as a symbol.

Now, as I have already said (repetition can be reassuring sometimes?) functions are sets of ordered pairs. This allows us to provide a meaning for the symbols we will use to represent functions. For instance, the following is a function: \[ f = \{(1, 12), (-1, 4), (3, 7)\} \] Now, there is no rhyme or reason to this function. It was invented just as I typed it. It is wholly arbitrary. But I am going to use it to introduce another key concept from this section, that of function notation.

Function Notation

The purpose of functions is to help write down ideas about input/output relationships. Function notation is one of the key ways we do it. Here is how it works. It starts with a symbol or name for a function, like so: \[ f \] Next, what follows are a pair of parentheses: \[ f() \] Now, in your mathematical past, parentheses may have been used to represent multiplication, no? I mean, in the expression \[ x(y + 2) \] the parentheses are somehow connected with multiplication—or, at least we know that an equivalent expression can be obtained by distributing or "multiplying the expression all out". But this is NOT what parentheses mean in the present scenario. It may seem annoying that mathematicians recycle symbols like that: they use the same symbols in different contexts to mean different things. Actually, without commenting on it, we also used parentheses in the section on interval notation to symbolize something totally different! Remember that there we might have written something like "\( (3, 5) \)" and what we meant was "the interval of \(x\)-values" that includes everything from 3 to 5 but not those endpoints"? How annoying is that when in still another context, the symbol \( (3, 5) \) might instead represent an ordered pair—which one might intend to graph on the \((x,y)\)-plane? So, you see, math symbols really do not have any meaning on their own. They must be given meaning by a wider discussion and context. So, we proceed with function notation. As I was saying, we have 1) the name of a function, say "\(f\)". Then 2) we write parentheses. Then, we place something inside the parentheses, like so: \[ f(3) \] and whatever goes inside the parentheses is the input. Indeed, these symbols must be accompanied by some clarification or context that communicates the fact that \(f\) is a function and that, therefore, whatever goes in the parentheses that follows the symbol \(f\) can be identified as the input. Now—we have to ask—what does the symbol, \(f(3)\) mean. It, too, has a meaning; everything in math has a very precise meaning! The symbol "\(f(3)\)" represents

The output you get from the function \(f\) when the input is 3.
For instance, if "\(f\)" is really our own private symbol for what the calculator labels "\(x^2\)"—and which we could also call "the squaring function"—then \(f(3) = 9\). On the other hand (recall that functions are sets of ordered pairs, where the first coordinate is input and the second coordinate is the output) if \(f\) represented my arbitrary function from above, \[ f = \{(1, 12), (-1, 4), (3, 7)\} \] then \[ f(3) = 7 \] Do you see how I came up with that? I simply looked up in a list of ordered pairs that defines \(f\), found the first coordinate (the input) 3 and then consulted the second coordinate to discover that the output is 7.

Now, math would be pretty limited if we could only consider input/output relations with only a finite number of possible inputs—or at least ones that we could eaily write down or list like this one: \[ f = \{(1, 12), (-1, 4), (3, 7)\} \] So, really, we won't be in the habit of doing that beyond this section—which we do here only for the purpose of illustrating the precise definition of function. Instead, if we are dealing with a really unlimited function—one which can take any number of inputs—then we have to use another way to define it. But we still use function notation to do it. Why? Because this notation (eventually) provides a handy way of writing down various ideas to be introduced in this—and, really—later math courses. So, to communicate that we intend to use the symbol "\(f\)" for the "square it" function, we write \[ f(x) = x^2 \] Of course, we will recycle the symbol \(f\) (sometimes resorting to \(g\) and \(h\) as symbols as needed) many times over in defining functions such as \[ f(x) = 6 - x^2 \] and \[ f(x) = \frac{x^2 - 2x + 1}{x + 1} \] The thing to conteplate in this section is just what do we mean by these expressions. What does the symbol "\(f(x)\)" mean in these expressions? Well, as I have said, \(f\) represents the name or symbol for a function—a set of ordered pairs. What set of ordered pairs does \[ f(x) = x^2 \] represent? It Well, functions are sets of "(input, output)" pairs. And, as I mentioned, when I write "\(f(\textrm{something})\)" you can be pretty darn sure that that "something" inside the parentheses is a symbol for an input for the function. Also, to reiterate, the whole symbol \[ f(\textrm{thing}) \] represents "the output from the function \(f\) when the input is 'thing'". So, what does it mean when I place a variable in as my input? Well, just as in other sections in which we discussed the variable \(x\), there is actually a secret narrative behind this that gives it meaning. What is really being conveyed goes something like this:

Let \(f\) be a function that takes any real number \(x\) as its input and whose output is found by doing the computation of \(x^2\) or "input-squared".
Since nobody wants to be that wordy (except me and a certain minority of mathematicians), we just write \[ f(x) = x^2 \] Put simply, this defines a rule by which we can compute the outputs of the function \(f\) for any given input. And this is really handier than having to consult a list of input/output pairs like we have to do with the definition \[ f = \{(1, 12), (-1, 4), (3, 7)\} \]

Domain and Range

There is just a tad more vocab we should introduce while we are on the subject of definitions and generality: those of domain and range. Every function has a domain (what is it?). Every function has a range. What do these words mean? Well, functions are about "input/output" relationships, as I hope you are beginning to know. Domain and range just have to do with them.

Definition:

The domain of a function is the set of all first coordinates in the function. Put another way, it is the "set of all allowable inputs."

Definition:

The range of a function is the set of all second coordinates in the function. Put another way, it is the "set of all outputs" that result from its inputs.

Our task now is just to practice all these fundamental definitions, and to practice function notation.

Recall that mathematicians call sets of ordered pairs "relations". Consider this (arbitrary) relation in answering the questions that follow: \[ f = \{(-5, 6), (3, 9), (-2, -8), (9, 6)\} \]

  1. Is \(f\) a function?

    Answer:

    Yes, \(f\) is a function. I know this because functions are sets of ordered pairs (relations) in which no first coordinate appears more than once.
  2. Identify the value of \(f(3)\).

    Answer:

    What goes inside the parentheses that follows the name of a function is the intended "input". In a function, the first coordinates are the inputs, and the second coordinates are the outputs. The symbol "\(f(3)\)" represents "the output from the function \(f\)" when the input is 3. Therefore, the answer is 9: \[ f(3) = 9 \]
  3. What is the domain of this function? Answer using set notation.

    Answer:

    The domain of a function is its set of inputs. So, the domain of \(f\) is \[ \{-5, -2, 3, 9\} \] I have listed from least to greatest here, but the order in which you list elements in a set doesn't matter.
  4. What is the range of this function? Use set notation for your answer.

    Answer:

    The range of a function is its set of outputs. So, the thing to do here is just to list the outputs (the second coordinates): \[ \{6, 9, -8\} \] Notice that I didn't list 6 twice! There's no need as 6 was already in "the set of outputs"; after all, 6 is 6, no? Put another way, repeating elements in a set is redundant.

Recall that mathematicians call sets of ordered pairs "relations". Consider this (arbitrary) relation in answering the questions that follow: \[ f = \{(-3, 6), (9, -1), (-2, -8), (9, 6)\} \] Is this relation a function?

Answer:

Nope! Notice that the first coordinate 9 is used twice. Therefore, this relation is not a function.

Let the function \(f\) be defined by the rule: \[ f(x) = 7 - x^2 \] Compute the indicated values.

  1. Find \(f(0)\)

    Answer:

    The symbol "\(f(0)\)" represents "the output from the function \(f\) when the input is \(x = 0\)". Put another way, all we have to do is "plug in" 0 for \(x\) in the formula that determines the outputs. \[ f(0) = 7 - 0^2 = 7 - 0 = 7 \] Simple as that! (These aren't meant to be hard, just to clarify the meaning of the notation and the concepts before we do more difficult things with them.)
  2. Find \(f(3)\)

    Answer:

    \[ f(3) = 7 - 3^2 = 7 - 9 = -2 \]
  3. Find \(f(-3)\)

    Answer:

    \[ f(-3) = 7 - (-3)^2 = 7 - 9 = -2 \] You have to mind your order of operations when you do these problems! Note that I first squared \(-3\), i.e., I did \[ (-3)^2 = (-3)(-3) = 9 \] Then I did the subtraction.

Let the function \(f\) be defined by the rule: \[ f(x) = 3x^2 + x - 2 \] Compute the indicated values.

  1. Find \(f(0)\)

    Answer:

    The symbol "\(f(0)\)" represents "the output from the function \(f\) when the input is \(x = 0\)". Put another way, all we have to do is "plug in" 0 for \(x\) in the formula that determines the outputs. \[ f(0) = 3\cdot 0^2 + 0 - 2 = 3 \cdot 0 - 2 = 0 - 2 = -2 \]
  2. Find \(f(2)\)

    Answer:

    \[ f(2) = 3\cdot 2^2 + 2 - 2 = 3 \cdot 4 + 0 = 12 \] Mind your order of operations!
  3. Find \(f(-2)\)

    Answer:

    \[ f(-2) = 3\cdot (-2)^2 + (-2) - 2 = 3 \cdot 4 - 2 - 2 = 12 - 4 = 8 \]
